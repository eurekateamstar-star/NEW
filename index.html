🚒 戰術指揮看板系統
獨立建置教學手冊

本系統採用純前端網頁（HTML/JS）開發，並結合 LINE LIFF (獲取使用者身分與發送圖卡) 與 Google Firebase (即時資料庫與雲端硬碟)。為確保貴單位的資料獨立與安全，請承辦人員依照以下 5 個步驟建立專屬的系統環境。

📍 步驟一：申請 Google Firebase (建立專屬資料庫)

這一步是為了建立儲存戰術看板資料的專屬空間，並取得系統需要的「金鑰」。

  1. 前往 Firebase Console，使用 Google 帳號登入。
  2. 點擊畫面上的 「新增專案 (Add Project)」，輸入專案名稱（例如：fire-tactical-board），一直按下一步直到建立完成。
  3. 進入專案總覽畫面後，在畫面正中央看到三個圓圈圖示 (iOS / Android / Web)，點擊第三個「網頁圖示 </>」 來新增一個網頁應用程式。
  4. 隨便輸入一個應用程式名稱 (例如：web-app)，並點擊「註冊應用程式」。
  5. 【這一步最重要】 註冊完成後，畫面上會出現一長串程式碼，請找到裡面以 const firebaseConfig = { ... } 開頭的這一段。
    👉 請把大括號 { 到 } 中間的這整段金鑰資料複製下來，先貼在旁邊備用！
  6. 接下來要建立資料庫。在左側選單找到 「Firestore Database」，點擊「建立資料庫」。選擇地點（建議選 asia-east1 台灣）。建立完成後，點擊上方的 「規則 (Rules)」 頁籤，將裡面的內容改為以下這段並點擊「發布」： 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
  7. 最後建立照片儲存空間。在左側選單找到 「Storage」，點擊「開始使用」。一樣點擊 「規則 (Rules)」 頁籤，將內容改為以下這段並發布： 
    rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if true;
    }
  }
}

📍 步驟二：申請 LINE LIFF (綁定 LINE 帳號)

這一步是為了讓系統能抓取使用者的 LINE 名字，並把戰術圖卡回傳到群組。

  1. 前往 LINE Developers 登入。
  2. 建立一個 Provider (服務提供者，可輸入單位名稱)。
  3. 在該 Provider 內，建立一個 「LINE Login」 的 Channel。
  4. 進入剛建好的 Channel，點擊上方的 「LIFF」 頁籤，點擊 「Add」 新增 LIFF APP。
    • Size: 選擇 FULL
    • Endpoint URL: 先隨便填一個 https://google.com，等一下步驟四網頁發布後再回來改。
    • Scopes: 務必勾選 profile (獲取名字) 與 chat_message.write (允許發送圖卡)。
  5. 建立完成後，你會得到一串 LIFF ID（格式大約是 1234567890-abcdefg）。請將這串 ID 複製下來備用。
  6. 【重要設定】點擊畫面上方的「LIFF」頁籤回到列表，往下捲動到底部，找到 shareTargetPicker 的開關，將它 開啟 (變成綠色)。這能讓系統發送漂亮的圖文小卡到群組！

📍 步驟三：修改原始碼 (將系統換成您的設定)

請將本手冊最下方的「附錄：系統完整程式碼」複製，貼到您電腦的「記事本」中，並修改以下三個地方：

  1. 替換資料庫金鑰：
    用 Ctrl+F 搜尋 firebaseConfig，將裡面的內容替換成您在 步驟一 (第 5 點) 複製下來的專屬金鑰。
  2. 替換 LIFF ID：
    用 Ctrl+F 搜尋 liffId:，替換成您在 步驟二 (第 5 點) 拿到的 ID。
  3. 替換分享卡片的網址：
    用 Ctrl+F 搜尋 uri: "https://liff.line.me/，把後面的 ID 也換成您專屬的 LIFF ID。這樣別人才點的進來您的系統。

修改完成後，請將記事本另存新檔，檔名命名為 index.html，檔案類型選擇「所有檔案」，編碼選擇「UTF-8」。

📍 步驟四：將網頁上架到 GitHub (免費網頁空間)

我們將使用 GitHub 提供的免費伺服器來放置你的網頁檔案。

  1. 前往 GitHub 註冊或登入帳號。
  2. 登入後，點擊左上角的 「New」 建立一個新的 Repository (儲存庫)。設定為 Public (公開)，然後點擊 Create repository。
  3. 在新畫面中，點擊 「uploading an existing file」 的連結。
  4. 把剛剛儲存好的 index.html 檔案拖曳進去，接著點擊綠色按鈕 「Commit changes」 上傳。
  5. 上傳成功後，點擊畫面上方的 「Settings (設定)」，在左側選單找到並點擊 「Pages」。
  6. 在「Branch」底下的選單，把 None 改成 main，然後點擊旁邊的 「Save」。
  7. 等待大約 1~2 分鐘，重新整理頁面，畫面最上方會出現一行字：
    Your site is live at https://你的帳號.github.io/專案名稱/。
    把這串網址複製起來。
🚨 【非常重要】：回到「步驟二」的 LINE Developers 後台，把你剛剛申請的 LIFF APP 的 Endpoint URL 替換成這個 GitHub 的網址並存檔！

📍 步驟五：系統開通與第一位管理員授權

恭喜！系統已經架設完成。接下來要進行「開光」手續。

  1. 拿起手機，把你的 LIFF 網址 (格式為 https://liff.line.me/你的LIFF-ID) 貼到任何一個 LINE 聊天室並點擊打開。
  2. 畫面會要求 LINE 授權，點擊「許可」。
  3. 點選畫面上方第四個頁籤 「管理指令」。
  4. 因為系統目前是全新的，沒有任何管理員，所以防呆機制會啟動。你會看到一顆綠色按鈕寫著 「➕ 系統無管理員 (點此綁定)」。
  5. 在「選擇成員」下拉選單選中你自己，然後點擊該按鈕。
  6. 設定完成！ 從現在開始，你就是這套系統的最高管理員，只有你可以看到「管理人員設定」區塊，並可以授權給其他長官或勤務中心人員。

        function extractWaterLines(data) {
            if (!data) return 0;
            let allText = Object.values(data).filter(val => typeof val === 'string' || typeof val === 'number').join(" ");
            if (!allText.trim()) return 0;
            
            let s = allText.replace(/['"]/g, '').replace(/１/g, '1').replace(/２/g, '2').replace(/３/g, '3').replace(/４/g, '4').replace(/５/g, '5').replace(/６/g, '6').replace(/一/g, '1').replace(/二/g, '2').replace(/兩/g, '2').replace(/三/g, '3').replace(/四/g, '4').replace(/五/g, '5');
            let match = s.match(/(\d+)\s*(?:條|股|線)?\s*水線/); 
            if (match) return parseInt(match[1]);
            match = s.match(/水線\s*[xX*]?\s*(\d+)/); 
            if (match) return parseInt(match[1]);
            if (s.includes("水線") && !s.includes("0條") && !s.includes("0水線") && !s.includes("水線0")) return 1; 
            return 0;
        }

        function createCardHtml(data) {
            let pStr = String(data.people || "0").replace(/['"人]/g, "") + "人";
            const side = data.side || "";
            let taskStr = (data.task || "未指派").replace(/['"]/g, "");
            const wLines = extractWaterLines(data); 
            
            if (wLines > 0 && !taskStr.includes("水線")) taskStr += ` <span style="color: var(--neon-blue); font-weight: 900;">(💧${wLines}線)</span>`;
            else if (wLines > 0) taskStr = taskStr.replace(/(\d+\s*(?:條|股|線)?\s*水線|水線\s*[xX*]?\s*\d+|水線)/g, '<span style="color: var(--neon-blue); font-weight: 900;">$&</span>');

            let time = "--:--";
            if (data.updateTime && data.updateTime.toDate) time = data.updateTime.toDate().toLocaleTimeString('zh-TW', { hour: '2-digit', minute: '2-digit', hour12: false });
            else if (data.updateTime && typeof data.updateTime === 'string') time = data.updateTime.substring(11, 16);

            let displayName = (data.team || "").replace(/['"]/g, "");
            const leaderStr = (data.leader || data.captain || "").replace(/['"]/g, "");
            if (leaderStr) displayName = displayName.replace("分隊", "") + leaderStr;

            const reporter = (data.reportedBy && data.reportedBy.name) ? data.reportedBy.name : "未知";
            let plateClass = "unit-plate"; 
            if (taskStr.includes("待命") || side.includes("集結") || side.includes("外部")) plateClass += " plate-standby";

            return `
                <div class="${plateClass}" onclick="showReporter('${reporter}', '${data.team}')">
                    <div class="plate-row"><span class="unit-name">${displayName}</span><span class="unit-pax">${pStr}</span></div>
                    <div class="plate-row"><span class="unit-task">${taskStr}</span><span class="unit-time">${time}</span></div>
                </div>`;
        }

        function getTargetZone(side) {
            if (!side) return '1';
            if (side.includes("內部") || side.includes("頂樓") || side.includes("火場")) return 'center';
            if (side.includes("2") || side.includes("二")) return '2';

            if (side.includes("4") || side.includes("四")) return '4';
            if (side.includes("集結") || side.includes("外部") || side.includes("待命")) return 'standby';
            return '1';
        }
